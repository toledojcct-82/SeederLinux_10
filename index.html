Recursos Principais

Tudo o que você precisa para provisionar estações Linux de forma eficiente e centralizada.

Multi-Organização

Gerencie múltiplas OMs com configurações independentes, cada uma com suas variáveis e identidade visual.

Offline-First

Scripts são gerados como bundles autônomos que podem ser executados mesmo sem conexão de rede ativa.

Variáveis Dinâmicas

Use placeholders {{VAR}} nos scripts que são automaticamente substituídos por valores reais.

Inventário OCS

Integração automática com OCS Inventory para coleta de informações das estações.

Painel Intuitivo

Interface web moderna e responsiva para gerenciar OMs, variáveis e gerar bundles.

Segurança

Autenticação por sessão, senhas criptografadas, proteção CSRF e SQL Injection.

Como Usar

Três passos simples para provisionar uma estação Linux.

1

Configure a OM

No painel administrativo, cadastre a organização e defina os valores das variáveis (IP do DC, proxy, etc).

2

Baixe o Agente

Transfira o agent.py para a estação Linux e configure a sigla da OM no arquivo.

3

Execute

Execute python3 agent.py e o provisionamento será feito automaticamente.
